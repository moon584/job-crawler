做一个爬虫项目，详细规划如下：

## 一、整体目标

搭建一个 Web 小系统（前后端分离或不分离），实现：

1. 前端表单填写爬取配置。
2. 后端根据配置发送 HTTP 请求，获取招聘接口返回的 JSON 数据。
3. 解析出职位列表（标题、类别等），并展示在页面上或打印到日志。

## 二、技术选型（简单轻量）

- 前端：HTML + CSS + JavaScript (原生或 Vue/React 简单版)
- 后端：Python + Flask（或 FastAPI）
- 请求库：`requests`
- 解析：根据 JSON 结构写简单的字段提取函数

## 三、详细步骤计划

### 阶段1：搭建后端 API

1. **创建 Flask 项目**

   - 安装 Flask：`pip install flask`
   - 新建 `app.py`

2. **设计接口**

   - `POST /crawl`

   - 接收 JSON 参数：

     json

     ```
     {
       "url": "https://careers.xxx.com/api/jobs",
       "method": "GET",
       "company_id": "C001"
     }
     ```

     

   - 校验参数（url 必须，method 默认 GET）

3. **实现爬取逻辑**

   - 使用 `requests.request(method, url, headers=headers)` 发送请求
   - 设置合理的 User-Agent
   - 捕获请求异常，返回错误信息

4. **实现解析逻辑（初版）**

   - 假设返回 JSON 格式，例如：

     json

     ```
     {
       "data": [
         {"jobId": "123", "title": "Java工程师", "category": "技术-研发"},
         ...
       ]
     }
     ```

     

   - 提取每个职位的 `title`、`category`、`external_id`（若有）

   - 因为不同公司结构不同，先写一个示例解析函数（后续可配置化）

5. **返回结果**

   - 将解析后的职位列表以 JSON 格式返回给前端：

     json

     ```
     {
       "code": 200,
       "jobs": [
         {"title": "...", "category": "..."}
       ]
     }
     ```

     

### 阶段2：搭建前端页面

1. **创建 HTML 页面**（`templates/index.html` 或静态文件）
   - 表单包含：
     - URL 输入框（必填）
     - Method 下拉选择（GET/POST）
     - 公司编号输入框（如 C001）
     - 提交按钮
2. **编写前端 JS**
   - 监听表单提交，阻止默认刷新
   - 使用 `fetch` 或 `axios` 发送 POST 请求到 `/crawl`
   - 接收后端返回的职位数据，动态展示在表格或卡片中
3. **展示结果**
   - 显示请求状态（成功/失败）
   - 显示解析出的职位数量及每个职位的标题、类别

### 阶段3：测试与调整

1. 找一个真实的招聘公开 API（例如一些公司公开的职位接口，注意合法性和 robots.txt）
2. 测试前端输入不同 URL，验证后端能正确请求并解析
3. 处理错误情况（超时、非 JSON 响应、字段缺失等）

### 阶段4（可选）：增加简单的解析配置

- 由于不同公司 JSON 结构不同，可以暂时在代码中用 `if company_id == 'C001'` 选择不同解析函数。
- 或者设计一个简易的 JSONPath 配置，让前端传入字段映射（如 `title_path="data[*].title"`），但这会增加复杂度，可后续再做。

## 四、不存数据库的具体实现要点

- 后端解析后直接返回 JSON，不调用任何数据库连接。
- 前端只展示结果，不发起存储请求。
- 日志记录爬取原始数据到文件（可选），便于调试。
